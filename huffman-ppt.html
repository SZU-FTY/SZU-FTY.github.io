David Huffman（1952）

MIT研究生，在一次课程作业中发明了构造最优前缀码的贪心算法

如何构造？→ 用二叉树来表示编码

构建哈夫曼树 — 输入

频率表：每个字符作为一个叶节点

5个独立节点，准备开始合并

构建哈夫曼树 — 算法

重复以下步骤，直到只剩一个节点：

  1. 从所有节点中取出频率最小的两个
  2. 合并为一个新节点，频率 = 两子节点之和
  3. 将新节点放回集合

这就是贪心策略：每次都合并最小的两个

使用最小堆（优先队列）可以高效实现

第1步：合并 C(1) + D(1)

取频率最小的两个：C(1) 和 D(1)

合并后：4个节点 — A(5), B(2), R(2), CD(2)

第2步：合并 B(2) + CD(2)

取频率最小的两个：B(2) 和 CD(2)

合并后：3个节点 — A(5), R(2), BCD(4)

第3步：合并 R(2) + BCD(4)

取频率最小的两个：R(2) 和 BCD(4)

合并后：2个节点 — A(5), RBCD(6)

第4步：合并 A(5) + RBCD(6)

最后合并：A(5) 和 RBCD(6) → 根节点(11)

完整哈夫曼树

左分支 = 0，右分支 = 1

从哈夫曼树到编码表

编码规则

从根节点到每个叶节点的路径：

  • 走左分支 → 记 0
  • 走右分支 → 记 1

路径上的 0/1 序列 = 该字符的编码

叶节点的路径互不为前缀 → 天然保证前缀码性质

解码过程

收到比特流：01101000111000111101100

解码规则：

  1. 从根节点出发
  2. 读到 0 → 走左，读到 1 → 走右
  3. 到达叶节点 → 输出字符，回到根节点
  4. 重复直到比特流结束

✓ 前缀码保证每次到达叶节点时解码唯一确定

哈夫曼编码的性质

最优性

定理：哈夫曼编码是最优前缀码

即：在所有前缀码中，WPL 最小

直觉解释：

  • 贪心策略每次合并最小的两个节点
  • 频率低的节点被推到树的最深处
  • 频率高的节点留在较浅的位置

哈夫曼树的唯一性

哈夫曼树不唯一，但 WPL 相同

原因：当多个节点频率相同时，合并顺序可以不同

例：第2步中 B(2)、R(2)、CD(2) 三者频率相同，可以选择不同的两个合并

结论：

  • 编码方案可能不同
  • 但所有方案的 WPL 相等，都是最优的

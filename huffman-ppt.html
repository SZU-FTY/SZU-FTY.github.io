编码问题：等长编码的浪费

ASCII编码：每个字符用 8位 表示

示例字符串：ABRACADABRA

核心挑战

💡 能否让高频字符用更短的编码？

→ 引出变长编码的思路

⚠️ 但是...

变长编码有歧义问题：

假设 A=0, B=01，收到 "001" 是 "AAB" 还是 "AB"？

变长编码的歧义 + 前缀码

反例：A=0, B=01

收到 001

是 "AAB"（0|0|01）还是 "AB"（0|01）？

❌ 无法确定！因为 "0" 是 "01" 的前缀

前缀码定义：任何字符的编码都不是另一个字符编码的前缀

✓ 前缀码可以无歧义解码

哈夫曼的贡献

贪心直觉：

  • 频率越高 → 编码越短
  • 频率越低 → 编码越长

David Huffman（1952）

MIT研究生，发明了构造最优前缀码的系统方法

→ 用二叉树表示编码，保证WPL最小

构建哈夫曼树 — 算法输入 + 步骤

输入：频率表（每个字符作为叶节点）

算法步骤（重复直到只剩一个节点）：

  1. 取频率最小的两个节点
  2. 合并为新节点，频率 = 两子节点之和
  3. 将新节点放回集合

构建演示（第1-2步）

第1步：C(1) + D(1) → CD(2)

→ 4个节点：A(5), B(2), R(2), CD(2)

第2步：B(2) + CD(2) → BCD(4)

→ 3个节点：A(5), R(2), BCD(4)

构建演示（第3-4步）

第3步：R(2) + BCD(4) → RBCD(6)

→ 2个节点：A(5), RBCD(6)

第4步：A(5) + RBCD(6) → 根(11) ✓

完整哈夫曼树

左分支=0，右分支=1

✓ 频率越高，路径越短，编码越短

为什么这样构建？

直觉解释：

  • 频率低的字符先合并 → 在树的深处 → 编码更长
  • 频率高的字符后合并 → 靠近根 → 编码更短

贪心策略的本质：

每次合并最小的两个节点，确保低频字符被"推"到树的最深处，从而获得最长的编码

生成编码表

解码过程

算法：

  1. 从根节点开始
  2. 读取一个比特：
    • 0 → 走左分支
    • 1 → 走右分支
  3. 到达叶节点 → 输出字符，回到根
  4. 重复直到比特流结束

无歧义性保证：

前缀码性质 → 每个字符对应唯一的叶节点 → 解码路径唯一

解码演示

比特流：0·110·10·0·1110·0·1111·0·110·10·0

逐步解码：

  • 读 0 → 左 → 到达A → 输出 A
  • 读 1 → 右，读 1 → 右，读 0 → 左 → 到达B → 输出 B
  • 读 1 → 右，读 0 → 左 → 到达R → 输出 R
  • 读 0 → 左 → 到达A → 输出 A
  • 读 1110 → 到达C → 输出 C
  • 读 0 → 到达A → 输出 A
  • 读 1111 → 到达D → 输出 D
  • 读 0 → 到达A → 输出 A
  • 读 110 → 到达B → 输出 B
  • 读 10 → 到达R → 输出 R
  • 读 0 → 左 → 到达A → 输出 A

结果：ABRACADABRA ✓

最优性

定理：哈夫曼编码是最优前缀码

即：在所有前缀码中，哈夫曼编码的加权路径长度（WPL）最小

直觉解释：

贪心策略确保：

  • 频率最低的字符在树的最深处（编码最长）
  • 频率最高的字符靠近根（编码最短）
  • 每一步都做出局部最优选择，最终达到全局最优

唯一性 + WPL计算

⚠️ 哈夫曼树不唯一

当存在频率相同的节点时，合并顺序可以不同

但所有哈夫曼树的WPL相同（都是最优的）

WPL计算：加权路径长度 = Σ(频率 × 编码长度)

时间复杂度

使用最小堆实现：

操作复杂度次数
建堆O(n)1次
取最小值O(log n)2×(n-1)次
插入新节点O(log n)n-1次

总复杂度：

O(n) + O(n log n) = O(n log n)

其中 n 是字符种类数

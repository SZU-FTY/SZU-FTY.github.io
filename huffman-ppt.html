核心挑战与哈夫曼的贡献

⚠️ 核心挑战：高频字符用短编码 → 变长编码有歧义问题

A=0, B=01 → "001" 是 "AAB" 还是 "AB"？

贪心直觉：频率越高 → 编码越短；频率越低 → 编码越长

David Huffman（1952）：MIT研究生，发明构造最优前缀码的系统方法

用二叉树表示编码，保证WPL最小

构建哈夫曼树 — 算法输入 + 步骤

输入：频率表（每个字符作为叶节点）

算法步骤（重复直到只剩一个节点）：

  1. 取频率最小的两个节点
  2. 合并为新节点，频率 = 两子节点之和
  3. 将新节点放回集合

构建演示（第1-2步）

第1步：C(1) + D(1) → CD(2)

→ 4个节点：A(5), B(2), R(2), CD(2)

第2步：B(2) + CD(2) → BCD(4)

→ 3个节点：A(5), R(2), BCD(4)

构建演示（第3-4步）

第3步：R(2) + BCD(4) → RBCD(6)

→ 2个节点：A(5), RBCD(6)

第4步：A(5) + RBCD(6) → 根(11) ✓

完整哈夫曼树

左分支=0，右分支=1

✓ 频率越高，路径越短，编码越短

为什么这样构建？

直觉解释：

  • 频率低的字符先合并 → 在树的深处 → 编码更长
  • 频率高的字符后合并 → 靠近根 → 编码更短

贪心策略的本质：

每次合并最小的两个节点，确保低频字符被"推"到树的最深处，从而获得最长的编码

生成编码表

从根到叶：左分支=0，右分支=1

解码过程

算法：

  1. 从根节点开始
  2. 读取一个比特：
    • 0 → 走左分支
    • 1 → 走右分支
  3. 到达叶节点 → 输出字符，回到根
  4. 重复直到比特流结束

无歧义性保证：

前缀码性质 → 每个字符对应唯一的叶节点 → 解码路径唯一

解码演示

比特流：0·110·10·0·1110·0·1111·0·110·10·0

逐步解码（从根出发，0=左，1=右，到达叶节点输出字符）：

0→A110→B10→R0→A
1110→C0→A1111→D0→A
110→B10→R0→A

结果：ABRACADABRA ✓

最优性与时间复杂度

定理：哈夫曼编码是最优前缀码

加权路径长度（WPL）最小

  • 频率最低 → 树最深 → 编码最长
  • 频率最高 → 靠近根 → 编码最短
  • 局部最优 → 全局最优

总复杂度：O(n log n)

操作复杂度次数
建堆O(n)1次
取最小值O(log n)2(n-1)
插入O(log n)n-1

n = 字符种类数

唯一性 + WPL计算

⚠️ 哈夫曼树不唯一

当存在频率相同的节点时，合并顺序可以不同

但所有哈夫曼树的WPL相同（都是最优的）

WPL计算：加权路径长度 = Σ(频率 × 编码长度)
